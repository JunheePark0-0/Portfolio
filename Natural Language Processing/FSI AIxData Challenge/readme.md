# 🛡️ FSI AIxData Challenge 2024: 개인정보 비식별화 및 위험 평가 서비스

## 📖 프로젝트 개요

**금융보안원(FSI)**이 주최한 AIxData Challenge 2024 출품작으로, 금융 데이터 내의 **개인정보를 자동으로 식별하여 마스킹(Masking)**하고, 공식 가이드라인에 따른 **위험 평가 리포트**를 제공하는 AI 솔루션입니다.

단순한 개체명 인식(NER)을 넘어, LLM(Solar)을 활용한 **RAG(검색 증강 생성)** 기반의 문서 검증(Groundness Check) 기능을 도입하였으며, 전체 프로세스를 웹 서비스로 구현하여 **금융보안원장상(최우수상)**을 수상했습니다.

## 🕐 개발 기간 
2024.07 ~ 2024.10 (4개월)

## ✨ 핵심 프로세스 및 전략

### 1. 모델 튜닝 및 고도화 (Model Tuning)
-   **Base Model 선정**: 다양한 한국어 PLM(`KLUE-BERT`, `KoELECTRA` 등)을 비교 실험하여 성능이 가장 우수한 `KLUE-BERT-base`를 선정했습니다.
-   **금융 도메인 Fine-tuning**: 일반적인 모델이 '대출', '보험', '카드사' 등의 금융 용어를 잘 인식하지 못하는 문제를 해결하기 위해, 네이버 지식iN과 은행 FAQ 데이터를 크롤링하여 추가 학습을 진행했습니다.

### 2. 데이터 처리 전략 (Data Processing)
-   **숫자 데이터 마스킹**: Tokenizer가 전화번호나 계좌번호를 분절하여 인식하지 못하는 한계를 극복하기 위해, **정규표현식(Regex)**으로 패턴을 먼저 마스킹(`[PAD]`)하고 모델 처리 후 복원하는 하이브리드 로직을 개발했습니다.
-   **데이터 증강**: 생성형 AI를 활용하여 부족한 금융 도메인 학습 데이터를 생성하고 학습에 활용했습니다.

### 3. 웹 서비스 및 RAG (Web & RAG)
-   **Groundness Check**: Solar LLM과 RAG를 활용하여, 모델이 생성한 위험 평가 결과가 실제 금융보안원 가이드라인에 부합하는지 팩트 체크를 수행합니다.
-   **리포트 생성**: 마스킹된 데이터의 재식별 위험도를 점수화하고, 이를 CSV 형태의 상세 리포트로 제공하는 기능을 웹 서비스로 구현했습니다.

## 🛠️ 주요 구성 요소

-   **`Model/Klue_BERT.ipynb`**: 금융 도메인 데이터로 Fine-tuning된 최종 NER 모델 학습 코드입니다. 정규식 기반의 전/후처리 로직이 포함되어 있습니다.
-   **`Model/Crawling.ipynb`**: 부족한 금융 데이터를 확보하기 위해 네이버 지식iN 등에서 텍스트를 수집하는 크롤러입니다.
-   **`Final Code.ipynb`**: 대회 제출용 최종 코드로, 입력 데이터를 받아 마스킹 처리를 수행하고 결과를 출력하는 전체 파이프라인을 담고 있습니다.
-   **`Trial/`**: 모델 선정 과정에서 시도했던 다양한 실험(BERT, LSTM 등) 기록이 남아있습니다.

## 🚀 실행 방법

1.  **환경 설정**: `torch`, `transformers` 등 딥러닝 및 NLP 관련 라이브러리를 설치합니다.
2.  **모델 학습 (선택)**: `Model/Klue_BERT.ipynb`를 실행하여 모델을 학습시킵니다. (학습된 가중치 파일 필요)
3.  **마스킹 실행**: `Final Code.ipynb`를 실행하고, 마스킹할 텍스트 데이터를 입력하면 비식별화된 결과를 확인할 수 있습니다.
